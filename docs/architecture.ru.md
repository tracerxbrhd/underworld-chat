# Заметки по архитектуре

## Базовая форма продукта

Система проектируется как backend-платформа сообщений с публичным лендингом для гостей и SPA-рабочим пространством диалогов для авторизованных пользователей.

## Идентичность и сессии

- Пользовательский логин и пароль как основа аккаунта
- Неизменяемый `public_id` как логин аккаунта
- Сессии устройств как главная единица доступа
- Recovery key и pairing flow для новых устройств
- Свободная регистрация с самим пользователем как центром выбора, кому писать и кому отвечать

## Дополнения доменной модели

- Редактируемый профиль: аватар, отображаемое имя, описание, дата рождения
- Заготовка личного канала заметок для переписки с собой
- Chat-модель, готовая и для 1:1 чатов, и для self-owned каналов
- Поиск, который сначала проходит по существующим чатам, а затем по подходящим пользователям без открытого диалога

## Мультиязычность

- Документация репозитория ведется на английском и русском
- Frontend использует словарь переводов и переключатель языка
- В Django включены `en` и `ru` через locale middleware
